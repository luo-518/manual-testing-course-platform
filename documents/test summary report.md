# 测试总结报告

---

# 1. 测试目的

本次测试针对“在线课程报名系统”进行功能测试，验证系统核心业务功能是否符合需求，包括课程展示、搜索、详情页、报名清单、订单提交、支付模拟与报名记录查询等。

目标是确保关键流程可用，输入校验合理，错误处理正确，系统可正常支持用户报名课程。

---

# 2. 测试范围

本次测试覆盖以下模块：

| 模块 | 说明 |
|------|------|
| 课程列表 | 列表展示、接口异常处理 |
| 搜索功能 | 关键字匹配、空输入校验 |
| 课程详情页 | 数据展示、资源加载异常 |
| 报名清单 | 加入/删除课程、名额校验 |
| 提交订单 | 登录验证、名额校验、清单校验 |
| 支付流程 | 支付成功/失败/超时 |
| 报名记录查询 | 正常显示、无记录、接口异常 |
| 手机号校验 | 数字格式校验、过滤无效输入 |

## 5.2 主要缺陷总结
以下为影响核心流程的高优先级缺陷：

1. **课程名额为 0 仍能加入报名清单（Critical）**  
   → 报名逻辑不符合业务规则。

2. **提交订单时名额校验未生效（Critical）**  
   → 存在违规报名成功风险。

3. **重复加入报名清单未提示（High）**  
   → 易导致重复订单和错误数据。

4. **未登录提交订单不跳转登录页（High）**  
   → 登录验证逻辑不完整。

详细缺陷请参见《bug report.md》。

---

# 6. 测试结论

整体而言，系统主要功能可运行，但存在影响关键流程的缺陷：

- 名额校验逻辑存在较大漏洞  
- 权限校验（未登录操作）不完全  
- 若干输入校验缺失（手机号、搜索关键词）  
- 部分页面资源加载错误未做容错处理  

系统在核心流程“加入清单 → 提交订单 → 支付”中存在重大缺陷，需要修复后才能进入下一轮测试或上线阶段。

---

# 7. 上线建议

**当前状态：不建议上线**  
原因：

- 存在阻断性缺陷（Critical）影响报名流程  
- 权限验证缺失导致错误操作可能绕过登录  
- 输入校验不严谨可能导致数据质量问题  

建议优先修复以下内容：

1. 名额校验逻辑（前端+后端）  
2. 重复加入报名清单判断  
3. 未登录流程跳转校验  
4. 输入校验（手机号、搜索框）  
5. 课程详情与资源加载异常处理
