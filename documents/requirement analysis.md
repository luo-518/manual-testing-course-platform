# 在线课程报名系统 – 需求分析（BRD/SRS）

## 1. 项目简介
在线课程报名系统用于提供课程浏览、搜索、报名、支付和记录查询等功能。本需求文档用于定义系统的核心业务需求，作为测试设计的依据。

---

## 2. 项目目标
- 支持用户在线浏览课程并查看详情  
- 支持课程报名、订单提交与支付模拟  
- 支持用户查看报名记录  
- 支持基础的校验（名额、手机号、登录状态）

---

## 3. 业务流程概述（高层流程）
1. 用户访问课程列表  
2. 用户搜索课程  
3. 查看课程详情  
4. 加入报名清单  
5. 提交订单  
6. 完成支付（模拟）  
7. 查看报名记录  

---

## 4. 功能需求（Functional Requirements）

### **R1：课程列表展示**
- 系统应展示所有可报名课程。
- 字段包括：课程名称、讲师、开课时间、价格、报名状态。

### **R2：课程搜索功能**
- 支持用户通过关键字搜索课程。
- 搜索结果在 2 秒内返回。

### **R3：课程详情页**
- 用户可进入详情页查看简介、课时数、评分、报名人数等信息。

### **R4：加入报名清单**
- 用户可以将课程加入报名清单。
- 系统需检查课程是否已满、是否已加入。

### **R5：提交报名订单**
- 用户可将清单中的课程提交订单。
- 需校验：是否登录、课程名额、价格一致性。

### **R6：支付模拟**
- 用户完成支付后订单状态应更新为“已报名”。
- 支付失败需提示错误。

### **R7：报名记录查询**
- 用户可查看所有历史报名记录。

### **R8：课程容量限制**
- 当课程名额为 0 时，应显示“已满”，并禁用加入按钮。

### **R9：登录校验**
- 未登录用户点击“报名清单”或“提交订单”时应跳转登录页。

### **R10：手机号格式校验**
- 报名时需绑定手机号，必须为 11 位数字。

## 5. 可测需求清单（Testable Requirements）
| 编号 | 可测点（Testable Requirement） |
|------|--------------------------------|
| TR1 | 课程列表正常返回与字段完整性 |
| TR2 | 课程搜索关键字校验 |
| TR3 | 课程详情页字段展示 |
| TR4 | 加入报名清单逻辑（未满、重复、未登录） |
| TR5 | 下单流程校验（登录、名额、清单） |
| TR6 | 支付结果处理（成功 / 失败 / 超时） |
| TR7 | 报名记录展示与异常处理 |
| TR8 | 名额边界值处理（0、1） |
| TR9 | 登录校验逻辑 |
| TR10 | 手机号输入校验 |

---

## 6. 非功能需求（可选）
- **响应时间**：搜索结果需在 2 秒内返回  
- **兼容性**：支持 Chrome、Firefox、Edge  
- **稳定性**：接口失败时需显示错误提示
